    Trying to access the /tiny endpoint we can see that it’s using some sort of file manager application.

    After some enumeration we saw that this file manager is public on github.
    Looking on the repository we found some default credentials that worked in our case ( https://github.com/prasathmani/tinyfilemanager )

    Looking at the bottom of the page we can see the version of the application.
    With a quick search on google we can find tons of exploits that we can use but I will do it manually just for the sake of it.

    In order to get a reverse shell you could download your reverse shell using curl or wget on the target machine and then execute it or you could simply just paste the reverse shell in the url.
